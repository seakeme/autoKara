## autoKara v1.0

第一个有正式安装包的版本。不懂 Python 的人也能直接装来用——双击 setup.exe 一路 Next 就行。

> 安装包已更新：修了一个会导致部分用户启动即崩的问题（英文发音词典 cmudict 在装机时偶尔下载失败，旧版会因此启动报 LookupError）。现在词典缺失只会让含英文的歌词降级为近似注音，不再影响启动，纯日文歌完全不受影响。已下过旧版的请重新下载安装包。

### 装法

下面附件里 `autoKara-setup.exe` 双击 → Next 装完。最后会弹出一个黑色 cmd 窗口配置环境（私有 Python、torch、模型，10–40 分钟，取决于网速 + 显卡），**别关它**，看到 "环境配置完成" 才算装好。

万一不小心关了也别紧张：下次双击桌面 autoKara 会自动检测到环境没装完、接着装。

国内网络拉得慢就用清华/阿里云镜像（具体一行命令在 README 里）。

### 这版相对于之前

**对齐**

默认改用分句约束对齐（phrase 模式）。歌词按行切、音频按 VAD 分句，DP 匹配后逐句单独对齐——这样不会再出现"开头几句飘到十几秒后"的整曲漂移。测了一首日文歌，平均置信度从 0.18 升到 0.41。每次跑完同时也跑整曲对齐做对照，按置信度自动挑分高的留下。

**注音**

- 修了个老 bug：像 "歌う" 这种"送假名跟汉字读音首字同形"的词，老版本会把汉字直接吞掉（"上手に歌う" 出 "上手にう"），现在正常了。
- 加了 `readings.txt`：自己加一行 `言う=いう` 之类，永久生效，重装也不会被覆盖。
- を 助词读音从 `wo` 修正成 `o`。
- 优先用 SudachiDict-full（条目更全），缺失自动回退 core。

**GUI**

- 多了个 "全自动模式" 复选框（默认开），跳过注音确认弹窗，全程不用点确认。
- 处理完不再只弹一个消息框：会出现一个带质量分、可疑行号、打开输出文件夹/复制路径按钮的面板。
- 高级参数会校验，输入错值会在日志里提醒并自动钳位，不再静默吞。
- 输出目录这些偏好会记忆（`%LOCALAPPDATA%\autoKara\settings.json`）。
- 处理中关窗会二次确认。

**额外的 QC 报告**

每次产出 `*.ass` 时另外写一份 `*.qc.txt`。顶部一行列出全曲置信度最低的 10% 行号——拿它去 Aegisub 优先听这几句就够了，不用全曲过一遍。

**装机/启动**

- Inno Setup 一键安装包，每用户安装无需管理员。
- 装 torch 时按 NVIDIA 驱动版本自动选 wheel：驱动 ≥570 走 cu128（RTX 50 系/Blackwell），≥525 走 cu121，没卡走 CPU。
- 配环境分阶段、可断点续装：哪步失败重跑就接着来。
- 启动器（launcher.py）先做依赖健康检查，缺什么会弹友好对话框 + 一键修复，不再是黑屏闪退。
- 每次运行写日志到 `%LOCALAPPDATA%\autoKara\logs`。
- 开始菜单加了"环境诊断"，命令行也是 `python main.py --doctor`，提 issue 时贴这个输出最快定位。

### 已知不足

- MMS-FA 在拖长音和有技巧的尾音上还是不完美。QC 报告里标红的行建议人工听一下。
- 模型缓存约 1.6 GB，第一次装好留在 `~\.cache\torch\hub\` 不会再下。
